## Условие лабораторной

Согласно ТЗ требовалось реализовать **3D-игру с космической тематикой**, включающую:

| Требование | Реализация                                                                                  |
|---|---------------------------------------------------------------------------------------------|
| 1–2 нетривиальные механики | Реализованы **две**: система нагрева инструмента + отталкивание астероидов и деформация меша |
| Cinemachine | Тряска камеры при ударе астероида (Impulse Source / Listener)                               |
| VFX Graph | Искры от сварки с непрерывной эмиссией                                                      |
| Light Baking | Запечённое глобальное освещение и отражения                                                 |
| Космическая тематика | Skybox-туманность, истребитель, астероиды                                                   |

## Игровой цикл

1. На корабле случайным образом появляется **2–3 трещин** в случайных позициях на корпусе. Каждая отмечена пульсирующим красным маяком.
2. На корабль начинают лететь астероиды (со случайными интервалами и количеством одновременно в воздухе). Каждый астероид светится оранжевым ореолом, чтобы игрок видел приближение.
3. У игрока в руках инструмент с двумя режимами:
   - **ЛКМ** — сварка: чинит трещины (наводись и держи)
   - **ПКМ** — отталкивание астероида в обратную сторону на 2 секунды
4. Любой режим **нагревает** инструмент. При перегреве (90% температуры) инструмент блокируется до охлаждения. Раздаётся звуковой сигнал тревоги.
5. **Победа** — все трещины запаяны.
6. **Поражение** — здоровье корпуса достигло нуля.

## Нетривиальные механики

### 1. Система нагрева инструмента

Файлы: `Assets/Scripts/Mechanics/HeatableObject.cs`, `Assets/Scripts/UI/HeatUI.cs`

Реализована независимая компонентная система:

- Нагреваемый объект имеет температуру `[0..100]` и порог перегрева
- Использование инструмента вызывает `ApplyHeat(rate)` — температура растёт пропорционально `Time.deltaTime`
- Без использования температура снижается с заданной скоростью охлаждения
- При достижении порога перегрева срабатывает `UnityEvent onOverheat`, инструмент блокируется до остывания ниже половины порога
- Визуальная обратная связь через **MaterialPropertyBlock** — эмиссия материала меняется по градиенту от тёмно-оранжевого до ярко-красного (без создания новых инстансов материала, GPU-friendly)
- UI-полоса нагрева слева внизу экрана, цвет переключается на красный при перегреве

### 2. Деформация мешей астероидов

Файл: `Assets/Scripts/Mechanics/DeformableMesh.cs`

При попадании луча сварки в астероид его меш физически прогибается:

- Скрипт работает с массивом вершин меша напрямую
- На каждый кадр луча: вершины внутри радиуса деформации сдвигаются по нормали с силой, обратно пропорциональной расстоянию до точки попадания
- Используется конвексный `MeshCollider` — необходимое условие для работы с динамическим Rigidbody
- Меш и коллайдер пересчитываются каждый кадр сварки

## Использованные встроенные инструменты Unity

### Cinemachine

`Assets/Scripts/Core/CameraShake.cs`

- `CinemachineBrain` на главной камере
- Виртуальная камера `CinemachineCamera` следует за `CameraRig` (дочкой Player), что позволяет MouseLook управлять направлением взгляда без конфликта с Cinemachine
- `CinemachineImpulseSource` на отдельном объекте генерирует импульс при удар астероида
- `CinemachineImpulseListener` на виртуальной камере вызывает плавную тряску

### VFX Graph

`Assets/WeldSparks.vfx`, интеграция в `Assets/Scripts/Tools/WeldingTool.cs`

- Spawn context — `Constant Rate` (50 частиц/сек) пока активен GameObject эффекта
- Initialize Particle — случайные lifetime, velocity (преимущественно вверх), size
- Output Particle — Additive blending для эффекта свечения
- Из кода: при попадании луча в трещину VFX позиционируется в точке попадания и активируется. При отпускании ЛКМ — деактивируется.

### Light Baking

- Корабль и Directional Light помечены как **Static** + **Contribute GI**
- Mode направленного света: `Mixed` (запечённые тени для статики + динамические для Player и астероидов)
- Environment Lighting Source: Skybox с пониженной интенсивностью (космос — тёмный)
- Запечён `Progressive CPU` лайтмаппером, разрешение 20 px/unit
